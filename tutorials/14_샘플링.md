## 1. 배경

이 예제는 OpenFOAM의 샘플링 유틸리티 사용법을 다룬다. 이 유틸리티를 사용하면 시뮬레이션 도중 또는 시뮬레이션이 끝난 후에 특정 면 또는 점에서의 데이터를 추출할 수 있다. 시뮬레이션 도중에 데이터를 샘플링하면 write-interval에 도달하기 전이라도 시뮬레이션의 진행 상황과 해의 적절성을 점검할 수 있다.

시뮬레이션 종료 후 후처리를 통해 데이터를 샘플링하려면 system/ 디렉토리의 sample 파일을 이용하고, 시뮬레이션 도중에 데이터를 샘플링하려면 controlDict 파일에 필요한 함수를 추가한다.

이 예제의 앞부분에서는 sample 파일과 controlDict 파일을 이용한 샘플링 방법을 다루며, 뒷부분에서는 OpenFOAM에서 사용할 수 있는 샘플링 종류를 다룬다.

## 2. 예제 개요

### 예제 구성

- 충격관(shock tube) 내부의 유동을 0.007초 간 해석한다.
- OpenFOAM의 샘플링 유틸리티를 이용해 시뮬레이션 도중에, 그리고 시뮬레이션 종료 후에 한 직선을 따라 데이터를 추출한다.

### 목표

- 샘플링의 기능 및 샘플링 유틸리티의 사용법을 이해한다.

### 데이터 처리

- 해석 결과를 ParaView로 가져와서 시각화하고, 샘플링 도구로 추출한 데이터를 분석한다.

## 3. 해석

### 3.1 전처리

> **Note:** OpenFOAM v2412:

> **Note:** OpenFOAM v2412: 0.orig/ 디렉토리의 사본을 생성하고 이름을 0으로 변경한다.

#### 3.1.2 system/sample 파일

sample 파일은 system/ 디렉토리에 위치한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
type sets;
libs            ("libsampling.so");

interpolationScheme     cellPoint;

setFormat       raw;

sets
(
    data
    {
        type            uniform;
        axis            x;
        start           (-4.995 0 0);
        end             (4.995 0 0);
        nPoints         1000;
    }
);

fields              (T mag(U) p);
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

`type`은 샘플링할 데이터의 종류를 정의하며, `sets` 또는 `surfaces`가 가능하다. `interpolationScheme`과 `setFormat`의 다양한 선택지는 뒷부분에서 다룬다.

`sets` 하위 딕셔너리의 각 데이터 집합에는 사용자가 임의로 이름을 부여하며, 이 예제에서는 단순히 `data`로 설정하였다. 데이터 집합의 괄호 안에는 아래의 항목을 정의한다.

- `type`: 샘플링 방법을 정의한다. 이 예제에서는 한 직선 상에 점을 균등하게 분포시키기 위해 `uniform`을 선택하였다.
- `axis`: 점의 좌표를 어떻게 출력할 것인지 정의한다. 이 예제에서는 `x`로 설정하여 각 점에 대해 x 좌표만 출력하도록 한다.
- `start`/`end`: 직선 샘플의 시작점과 끝점을 정의한다.
- `nPoints`: 직선 상의 점 개수를 정의한다.

`fields`에서는 어떤 변수를 샘플링할 것인지 정의한다.

정리하면, 두 점 (-4.995 0 0), (4.995 0 0)을 잇는 선분 위 등간격으로 1000개 지점에서 온도(T), 속력(mag(U)), 압력(p)의 값을 샘플링한다.

> **Note:** OpenFOAM v1906: `type`: `lineUniform`

> **Note:** OpenFOAM v2412: `libs`: `(sampling)`

#### 3.1.3 system/controlDict 파일

매 write-interval이 아닌 매 시간 단계마다 데이터를 샘플링하거나 솔버가 실행되는 동안 데이터를 샘플링하려면, sample 딕셔너리 대신 controlDict 파일에 함수를 추가해야 한다.

여기서는 shockTube 예제의 controlDict 파일을 수정하여, 앞에서 정의한 직선 샘플링이 시뮬레이션 실행 중에 매 시간 단계마다 수행되도록 설정한다.

controlDict 파일의 `functions` 하위 딕셔너리 끝에 아래의 코드를 추가한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
functions
{
    #includeFunc mag(U)

    linesample
    {
        type                sets;
        functionObjectLibs  ("libsampling.so");
        writeControl        timeStep;
        outputInterval      1;

        interpolationScheme cellPoint;

        setFormat       raw;

        sets
        (
            data
            {
                type        uniform;
                axis        x;
                start       (-4.995 0 0);
                end         (4.995 0 0);
                nPoints     1000;
            }
        );

        fields          (T mag(U) p);
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

`linesample`은 샘플링 도구의 설정을 담는 하위 딕셔너리이다. 이 이름은 임의의 이름으로 변경할 수 있으며, 솔버 실행 후 postProcessing 디렉토리 안에서 폴더 이름으로 사용된다.

`linesample` 하위 딕셔너리의 각 항목은 아래와 같다.

- `type`: `sets` 또는 `surfaces`를 선택할 수 있다. 더 다양한 종류는 뒷부분의 **4. 샘플링 종류**에서 다룬다.
- `functionObjectLibs`: 샘플링 작업에 필요한 기능을 제공한다.
- `writeControl`: 샘플링 데이터를 저장할 간격을 정의한다. `timeStep`인 경우 `outputInterval`에 따라 시간 단계 단위로 데이터가 저장된다. `outputInterval`이 1이면 매 시간 단계마다 샘플링 데이터를 저장하고, 2로 변경하면 2시간 단계마다 데이터를 저장한다.

### 3.2 시뮬레이션 실행

작업 디렉토리에서 터미널에 아래의 명령어를 입력하여 시뮬레이션을 실행한다.

```bash
blockMesh
setFields
sonicFoam
```

> **Note:** OpenFOAM v2412에서는 `sonicFoam` 대신 `rhoPimpleFoam`을 사용한다.

### 3.3 후처리

`sonicFoam` 솔버 실행이 끝난 후에는, 샘플링 방식에 따라 아래의 단계를 수행한다.

#### 3.3.1 sample 파일을 이용한 경우

sample 명령어를 이용하여 샘플 데이터를 추출한다.

```bash
postProcess -func sample
```

작업 디렉토리에 postProcessing 폴더가 생성되며, 그 안에 sample 폴더가 생성된다. 이 폴더에는 각 write-interval 별 폴더로 분리되어 모든 샘플링 데이터가 저장된다.

```bash
ls postProcessing/sample
```

아래는 0.007초 시점에서 샘플링 도구로 추출한 온도, 속력, 압력 분포를 나타낸다. (postProcessing/sample/0.007/)

![0.007초 시점에서 샘플링 도구로 추출한 데이터](../figures/14_sampling_result.jpeg)

> **Note:** ParaView의 `Plot Over Line` 필터는 *시각화 도구가 사용자 정의 선 위에서 격자 데이터를 보간*하는 방식이다. 반면에 이번 챕터에서 소개한 `sample` 함수는 *솔버가 자기 격자 위에서 직접 보간*해 디스크에 기록한다. 두 방법은 보간 시점과 위치가 다르므로 결과가 미세하게 다를 수 있고, 후자는 시간 단계마다 자동으로 누적되어 후처리 자동화에 유리하다. 그래프 비교가 목적이면 `Plot Over Line`이 빠르고, 시계열 정량 분석이 목적이면 본 챕터의 `sample`이 적합하다.

#### 3.3.2 controlDict 파일을 이용한 경우

솔버 실행 후 postProcessing 디렉토리와 그 하위 디렉토리가 생성된다. 매 write-interval이 아닌 매 시간 단계마다 폴더가 생성됨을 확인할 수 있다.

```bash
ls postProcessing/linesample
```

## 4. 샘플링 종류

샘플링에는 크게 두 가지 종류가 있다. 하나는 위 예제에서 사용한 `sets` 방식이고, 다른 하나는 `surfaces` 방식이다.

`sets` 방식에서는 앞에서 사용한 직선 샘플링과 같이 점에 대해 다양한 방식의 샘플링이 가능하며, 일종의 클라우드(cloud) 샘플링도 포함된다. `surfaces` 방식에서는 패치 근처의 면이나 한 점과 법선 벡터로 정의된 평면 위와 같이, 면 전체에 대한 샘플링이 가능하다.

먼저 두 방식의 공통점을 살펴보자. controlDict 파일 안에서 샘플링이 일어나는 경우, 두 방식 모두 위의 **3.1.3** 절에서 다룬 4개의 항목이 정의되어야 한다. 두 방식 모두 추가로 보간 기법(interpolation scheme)을 정의해야 하는데, 여기서는 `cell`과 `cellPoint` 두 가지 기법만 다룬다. `cell` 방식은 한 격자 셀 내부에서 셀의 중심에서의 값이 일정하다고 가정하며, `cellPoint` 방식은 셀의 중심과 꼭지점에서의 값 사이에서 선형 보간을 수행한다. 마지막으로, `fields` 괄호는 두 방식에서 동일한 형태이다.

### 4.1 sets

모든 sets 방식은 `setFormat`을 정의해야 한다. 예를 들어 csv 형식은 `interpolationScheme` 다음에 포함될 수 있다. 그 다음에 `sets` 하위 딕셔너리가 시작되며, 이 안에는 임의의 이름을 가진 괄호가 있다. 이 괄호 안에 sets 샘플링의 종류와 샘플링 점의 기하학적 위치가 정의된다. 아래에서 몇 가지 샘플링 종류를 다룬다.

#### 4.1.1 uniform

위의 예제에서 사용된 방식이다. 시작점에서 끝점까지의 직선 상에 일정한 간격으로 균등하게 분포된 정해진 개수의 점에서 샘플링을 수행한다. `axis` 항목은 출력 파일에 어떤 점 좌표를 기록할 것인지를 결정한다. `distance`로 설정하는 경우, 시작점과 샘플링 점 사이의 거리만 파일에 기록된다.

```cpp
lineX1
{
    type            uniform;
    axis            distance;

    start           (0.0201 0.05101 0.00501);
    end             (0.0601 0.05101 0.00501);
    nPoints         10;
}
```

#### 4.1.2 face

이 방식 역시 정의된 시작점에서 끝점까지의 직선 상에서 샘플링을 수행하지만, 직선이 자르는 격자 면마다 로그 파일에 기록한다.

```cpp
lineX2
{
    type            face;
    axis            x;

    start           (0.0001 0.0525 0.00501);
    end             (0.0999 0.0525 0.00501);
}
```

#### 4.1.3 cloud

`cloud` 방식은 `points` 괄호로 정의된 특정 점에서 데이터를 샘플링한다.

```cpp
somePoints
{
    type            cloud;
    axis            xyz;
    points          ((0.049 0.049 0.00501)(0.051 0.049 0.00501));
}
```

#### 4.1.4 patchSeed

`patchSeed` 방식은 `wall` 종류의 패치를 샘플링하는 데 사용된다. 예를 들어 벽면에서의 표면 흡착을 샘플링할 때 사용할 수 있다.

```cpp
patchSeed
{
    type            patchSeed;
    axis            xyz;
    patches         (".*Wall.*");
    maxPoints       100;
}
```

> **Note:** `patches`로는 `wall` 종류의 패치만 사용할 수 있다. 다른 종류의 패치를 입력하면 아무것도 샘플링되지 않으며 에러 메시지도 출력되지 않는다.

### 4.2 surfaces

모든 surfaces 방식은 `surfaceFormat`을 정의해야 한다. 실용적인 형식으로는 ParaView로 열 수 있는 vtk 형식과 gnuplot에서 사용할 수 있는 raw 형식이 있다. `sets` 괄호 대신 `surfaces` 괄호를 사용하며, `type`은 당연히 `surfaces`이다. `surfaces` 괄호 안에서는 각각 자체 괄호를 가진 임의의 개수의 면을 샘플링할 수 있다. 면 샘플링 종류 중 아래의 예제에 나오는 `plane`와 같은 종류는 뒷부분에서 다룬다.

```cpp
type            surfaces;

interpolationScheme cellPoint;
surfaceFormat   vtk;

fields
(
    U
);

surfaces
(
    yoursurfacename
    {
        type            plane;
        basePoint       (0.1 0.1 0.1);
        normalVector    (0.1 0 0);
        triangulate     false;
    }
);
```

#### 4.2.1 plane

`plane` 종류는 `basePoint`에 원점을 두고 `normalVector`에 수직인 평면을 생성한다. 이 `normalVector`는 `basePoint`가 아닌 원점을 시작점으로 한다. 면의 삼각형화(triangulation)를 끄려면 `triangulate false`를 추가한다.

```cpp
constantPlane
{
    type            plane;       // 항상 삼각형화됨
    basePoint       (0.0501 0.0501 0.005);
    normalVector    (0.1 0.1 1);

    //- Optional: 특정 zone에 한정
    // zone         zone1;

    //- Optional: 삼각형화 비활성화 (다각형을 지원하는 surfaceFormat에서만 가능)
    // triangulate  false;
    // interpolate  true;
}
```

`basePoint`와 `normalVector`의 원점은 `coordinateSystem`을 통해 설정할 수도 있다.

```cpp
coordinateSystem
{
    origin          (0.0501 0.0501 0.005);
}
```

#### 4.2.2 patch

`patch` 방식은 한 패치 전체에 대한 데이터를 샘플링한다. 문법은 `patchSeed`와 같아 보이지만, `wall` 종류가 아닌 패치도 사용할 수 있다. 기본적으로 면은 삼각형화되며, `triangulate false`로 끌 수 있다.

```cpp
walls_interpolated
{
    type            patch;
    patches         (".*Wall.*");
    //interpolate   true;
    // Optional: faces (default) 또는 삼각형화 여부
    // triangulate  false;
}
```

#### 4.2.3 patchInternalField

`patch` 방식과 유사하지만, 이 방식은 패치에서 법선 방향으로 일정 거리 떨어진 곳의 데이터를 샘플링한다 (`offsetMode normal`). 다른 방식으로도 거리를 정의할 수 있으며, 코드의 주석 부분에서 확인할 수 있다.

> **Note:** 샘플링은 패치 자체가 아닌 패치에서 일정 거리 떨어진 위치에서 수행되지만, 출력 파일에 기록되는 샘플 값의 기하학적 위치는 패치의 위치로 기록된다.

기본 삼각형화는 `triangulate false`로 끌 수 있다.

```cpp
nearWalls_interpolated
{
    // 패치 외부에서 셀 값을 샘플링한다.
    // 인접 셀일 필요는 없으며, 임의의 거리에 위치할 수 있다.
    type            patchInternalField;
    patches         (".*Wall.*");
    interpolate     true;

    // Optional: 패치 면 중심에서 샘플 점을 얻는 방법
    // (기본값은 'normal')
    //
    // //- 법선 방향 거리 지정
    offsetMode      normal;
    distance        0.1;
    //
    // //- 단일 균일 오프셋
    // offsetMode   uniform;
    // offset       (0 0 0.0001);
    //
    // //- 패치 면별 오프셋
    // offsetMode   nonuniform;
    // offsets      ((0 0 0.0001) (0 0 0.0002));

    // Optional: faces (default) 또는 삼각형화 여부
    // triangulate  false;
}
```

#### 4.2.4 triSurfaceSampling

`triSurfaceSampling` 방식은 stl 파일로 제공되는 평면에서 데이터를 샘플링한다. stl 파일을 만들려면 아래의 명령어를 사용해 패치 중 하나 (또는 그 이상) 으로부터 stl 파일을 생성할 수 있다.

```bash
surfaceMeshTriangulate name.stl -patches "(yourpatch)"
```

여기서 `yourpatch`는 constant/polyMesh/boundary 파일에 정의된 패치 중 하나의 이름으로 대체한다. `-patches` 옵션 없이 명령어를 실행하면 전체 격자 경계에 대한 stl 파일이 생성된다. 다음으로 constant/ 디렉토리에 triSurface/ 디렉토리가 없으면 새로 만들고, 생성한 stl 파일을 그 안으로 복사한다. 코드에서는 이 stl 파일을 표면으로 지정해야 한다. stl 파일이 패치 중 하나에서 추출한 것이라면 `source`로 `boundaryFaces`를 선택하는 것이 좋다.

```cpp
triSurfaceSampling
{
    // triSurface 상의 샘플링
    type            sampledTriSurfaceMesh;
    surface         integrationPlane.stl;
    source          boundaryFaces;
    // 무엇을 샘플링할 것인가:
    //   cells (가장 가까운 셀)
    //   insideCells (셀 내부의 삼각형만)
    //   boundaryFaces (가장 가까운 경계면)
    interpolate     true;
}
```

> **Note:** 대부분의 CAD 소프트웨어에서 3차원 도면의 면을 stl 파일로 내보낼 수 있다.

#### 4.2.5 isoSurface

`isoSurface` 방식은 본 예제에서 다룬 다른 샘플링 종류와는 다소 다른 방식이다. 지금까지 다룬 샘플링 종류는 모두 공간 상에 일정한 위치를 가지며, 시간에 따라 그 위치에서 변화하는 변수 값을 추출하는 방식이었다. 반면 `isoSurface` 방식에서는 공간 내에서 특정 값을 갖는 위치를 추적한다. 아래의 예제는 shockTube 예제의 sample 파일에 복사할 수 있다(물론 surfaces 종류 샘플링에 필요한 다른 옵션도 함께 필요하다). `surfaceFormat`을 `vtk`로 설정하면 공간 상에서 이동하는 충격파를 시각화할 수 있다. 샘플링의 vtk 파일과 전체 shockTube 케이스 모두를 ParaView로 함께 열어 결과를 비교할 수 있다.

> **Note:** `isoField`는 반드시 `scalarField`여야 한다.

```cpp
interpolatedIso
{
    // 보간된 값에 대한 등치면
    type            isoSurface;
    // 항상 삼각형화됨
    isoField        p;
    isoValue        9e4;
    interpolate     true;

    //zone          ABC;
    // Optional: zone에 한정
    //exposedPatchName fixedWalls;
    // Optional: zone에 한정

    // regularise   false;
    // Optional: 단순화하지 않음

    // mergeTol     1e-10;
    // Optional: 점을 합칠 격자 경계 상자의 비율
    // (기본값=1e-6)
}
```

#### 4.2.6 isoSurfaceCell

`isoSurfaceCell` 방식은 위에서 다룬 `isoSurface` 방식과 매우 비슷하지만, 이 방식은 등치면이 셀을 가로지르지 않고 값을 보간하지도 않는다.

```cpp
constantIso
{
    // 상수 값에 대한 등치면.
    // 삼각형이 셀을 가로지르지 않도록 보장한다.
    type            isoSurfaceCell;
    // 항상 삼각형화됨
    isoField        rho;
    isoValue        0.5;
    interpolate     false;
    regularise      false;
    // 단순화하지 않음

    // mergeTol     1e-10;
    // Optional: 점을 합칠 격자 경계 상자의 비율
    // (기본값=1e-6)
}
```
