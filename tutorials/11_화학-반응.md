## 1. 배경

화학 반응을 모델링하는 데 사용되는 일반적인 두 가지 접근법은 아래와 같다.

### 1.1 부분 교반 반응기(PaSR) 모델

부분 교반 반응기(Partially Stirred Reactor, PaSR) 모델은 연소와 같이 열역학적·화학적 반응을 수치적으로 모델링하기 위한 모델이다. PaSR 방식에서는 격자 셀을 두 개의 영역으로 분할한다. 하나는 반응이 일어나는 영역(reacting zone)이고, 다른 하나는 반응이 일어나지 않는 영역(non-reacting zone)이다. 반응 영역은 완전 교반 반응기(perfectly stirred reactor, PSR)로 모델링되며, 모든 반응물이 서로 완전히 혼합되어 있다고 가정한다.

반응기에서 관심을 가지는 농도는 아래와 같이 세 가지이다.

1. 공급 흐름에서 주요 성분의 평균 농도 $c_{in}$
2. 반응 영역에서 혼합물의 농도 $c$
3. 반응기 출구에서의 농도 $c_{exit}$

반응 영역에서 반응은 시간 $\tau_c$ 동안 일어나며, 혼합물의 농도는 $c_{in}$에서 $c$로 변화한다. 반응이 일어나지 않는 영역에서는 반응이 일어난 혼합물이 시간 $\tau_{mix}$ 동안 반응이 일어나지 않은 혼합물과 섞이고, 최종 출구 농도 $c_{exit}$가 결정된다.

이 모델에서 계산해야 할 핵심 파라미터는 반응 속도이며, 반응 속도는 화학 반응 시간과 반응기 내 전체 전환 시간(반응 시간 + 혼합 시간)의 비에 비례한다.

$$\kappa_k = \frac{\tau_c}{\tau_c + \tau_{mix}}$$

### 1.2 Eddy Dissipation Concept (EDC) 모델

EDC 모델은 반응과 난류의 상호작용에 주목하며, 전체 반응 속도가 난류 혼합에 의해 결정된다고 보는 모델이다. 다양한 연소 환경에 대해 폭넓게 적용되며 우수한 결과를 보이기 때문에, 연소 모델링에 널리 사용된다.

이 모델은 대부분의 반응이 미세 난류 구조(fine turbulence structure) 내에서 일어난다고 가정하며, 이 미세 구조를 완전 교반 반응기로 모델링한다. 반응 질량 분율과 미세 구조와 주변 유체 사이의 물질 전달 속도를 모델 입력값으로 필요로 한다.

미세 구조가 차지하는 질량 분율 $\gamma^{*}$는 아래와 같이 표현된다.

$$\gamma^{*} = \left(\frac{u^{*}}{u'}\right)^2$$

여기서 $u^{*}$는 미세 구조의 질량 평균 속도이다. 미세 구조는 난류 운동 에너지 $u'^2$가 거의 일정한 영역에 존재한다.

미세 구조와 주변 유체 사이의 단위 유체 부피·단위 시간 당 물질 전달 속도는 아래와 같이 모델링된다.

$$\dot{m} = 2 \cdot \frac{u^{*}}{L^{*}} \cdot \gamma^{*}$$

여기서 $L^{*}$는 미세 구조의 특성 길이이다.

## 2. 예제 개요

### 예제 구성

- `reactingFoam` 솔버를 이용하여 혼합 엘보(mixing elbow) 내에서 $CH_4$와 $O_2$의 연소를 해석한다.
- 해석 영역 초기 상태: $N_2$로 가득 차 있음
- velocity-inlet-5
  - 속도: 1 m/s
  - 질량 분율: 23% $O_2$, 77% $N_2$ (공기 투입)
  - 온도: 800 K
- velocity-inlet-6
  - 속도: 3 m/s
  - 질량 분율: 50% $CH_4$, 50% $N_2$ (연료 투입)
  - 온도: 293 K
- 작동 압력: 1 bar
- 작동 온도: 298 K
- 단열 벽면

### 목표

- OpenFOAM에서의 다성분(multi-species) 및 반응 모델링을 이해한다.

### 데이터 처리

- 해석 결과를 ParaView로 가져와서 확인한다.

## 3. 해석

### 3.1 전처리

#### 3.1.1 예제 복사

아래의 예제를 작업 디렉토리에 복사한다.

```bash
$FOAM_TUTORIALS/combustion/reactingFoam/laminar/counterFlowFlame2D
```

아래의 경로로부터 예제에서 사용할 GAMBIT 격자를 작업 디렉토리로 복사한다.


GAMBIT으로 생성한 격자는 OpenFOAM과 바로 호환되지 않기 때문에, 아래의 명령어를 이용하여 OpenFOAM의 격자 파일로 변환해야 한다.

```bash

```

OpenFOAM 형식으로 변환된 격자는 constant/ 디렉토리 하위에 새로 생성된 polyMesh/ 디렉토리에 저장된다. paraFoam 명령어를 이용해 paraview로 형상 및 격자를 확인한다.

```bash
paraFoam
```

#### 3.1.2 0/ 디렉토리

새로운 경계 조건에 맞춰 0/ 디렉토리의 모든 파일을 수정한다. 경계면의 이름과 `type`은 constant/polyMesh/ 디렉토리의 boundary 파일에 저장된다. 예를 들어, U 파일은 아래와 같이 수정한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
dimensions      [0 1 -1 0 0 0 0];

internalField   uniform (0 0 0);

boundaryField
{
    wall-4
    {
        type            fixedValue;
        value           uniform (0 0 0);
    }
    velocity-inlet-5
    {
        type            fixedValue;
        value           uniform (1 0 0);   // 1 m/s
    }
    velocity-inlet-6
    {
        type            fixedValue;
        value           uniform (0 3 0);   // 3 m/s
    }
    pressure-outlet-7
    {
        type            zeroGradient;
    }
    wall-8
    {
        type            fixedValue;
        value           uniform (0 0 0);
    }
    frontAndBackPlanes
    {
        type            empty;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

이 시뮬레이션에서 일어나는 반응은 $CH_4$와 $O_2$가 연소되어 $CO_2$와 $H_2O$를 생성하는 반응이다. $N_2$는 비반응성 성분이다. 모든 화학종에 대한 경계 조건과 초기값은 0/ 디렉토리에서 설정해야 한다. 이 값들은 질량 분율로(0과 1 사이의 무차원 값), 예를 들어 N2 파일은 아래와 같이 설정한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
dimensions      [0 0 0 0 0 0 0];

internalField   uniform 1.0;                // initially N2 filled

boundaryField
{
    wall-4
    {
        type            zeroGradient;
    }
    velocity-inlet-5
    {
        type            fixedValue;
        value           uniform 0.77;       // N2 mass fraction 0.77 at air inlet
    }
    velocity-inlet-6
    {
        type            fixedValue;
        value           uniform 0.5;        // N2 mass fraction 0.5 at fuel inlet
    }
    pressure-outlet-7
    {
        type            zeroGradient;
    }
    wall-8
    {
        type            zeroGradient;
    }
    frontAndBackPlanes
    {
        type            empty;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** 0/ 디렉토리에 특정 성분에 대한 파일이 존재하지 않는 경우, Ydefault 파일의 값이 사용된다.

#### 3.1.3 constant/ 디렉토리

thermophysicalProperties 파일에서 각 성분의 물리적 특성을 설정한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
thermoType
{
    type            hePsiThermo;
    mixture         reactingMixture;
    transport       sutherland;
    thermo          janaf;
    energy          sensibleEnthalpy;
    equationOfState perfectGas;
    specie          specie;
}

inertSpecie N2;

chemistryReader foamChemistryReader;

foamChemistryFile     "$FOAM_CASE/constant/reactions";

foamChemistryThermoFile "$FOAM_CASE/constant/thermo.compressibleGas";
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

`mixture` 항목은 혼합 물성과 열용량을 계산하기 위해 `reactingMixture`로 설정한다. 열용량은 janaf 다항식(janaf polynomial)을 통해 계산된다.

`inertSpecie`로 $N_2$가 정의되어 있다. OpenFOAM의 반응 솔버에서, 비반응 성분은 질량 보존을 만족시키기 위해 질량 균형 방정식을 통해 계산된다.

$$\text{비반응 성분의 질량 분율} = 1 - \sum \text{다른 모든 성분의 질량 분율}$$

성분과 반응은 `foamChemistryFile`을 통해 정의되며, 이 시뮬레이션에서 성분과 반응은 constant/ 디렉토리의 reactions 파일로부터 읽어들인다.

```cpp
species
(
    O2
    H2O
    CH4
    CO2
    N2
);

reactions
{
    methaneReaction
    {
        type        irreversibleArrheniusReaction;
        reaction    "CH4 + 2O2 = CO2 + 2H2O";
        A           5.2e16;
        beta        0;
        Ta          14906;
    }
}
```

> **Note:** OpenFOAM v1906: reactions 파일에 원소(elements)들도 함께 나열되며, 계산 과정에서 원소 균형 계산이 수행된다.

> **Note:** OpenFOAM v2412: reactions 파일에 원소(elements)들도 함께 나열되며, 계산 과정에서 원소 균형 계산이 수행된다.

`species` 하위 딕셔너리에서 화학종 ($O_2$, $H_2O$, $CH_4$, $CO_2$, $N_2$)을 정의한다. `reactions` 하위 딕셔너리에서는 반응의 종류를 비가역 Arrhenius 반응(`irreversibleArrheniusReaction`)으로 정의한다.

[예제 2](./02_내장-격자-생성-도구-blockMesh.md)에서는 균질 혼합물(homogeneous mixture)인 기체 혼합물의 물성을 계산하는 계수가 `mixture` 하위 딕셔너리에서 정의되었다. 그러나 이 예제에서 혼합물은 비균질하기 때문에, 각 셀의 조성에 따라 혼합물의 물성을 계산하기 위해서는 각 성분의 물성을 개별적으로 계산할 수 있는 계수가 필요하다. 각 성분의 계수는 `foamChemistryThermoFile`로 정의되며, 이는 constant/ 디렉토리의 thermo.compressibleGas 파일을 읽어들인다. 이 단계는 thermophysicalProperties 파일에 명시되어 있다. 예를 들어, 각 모델에 대한 $O_2$의 계수는 아래와 같다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
O2
{
    specie
    {
        molWeight       31.9988;
    }
    thermodynamics
    {
        Tlow            200;
        Thigh           5000;
        Tcommon         1000;
        highCpCoeffs    ( 3.69758 0.00061352 -1.25884e-07 1.77528e-11
                          -1.13644e-15 -1233.93 3.18917 );
        lowCpCoeffs     ( 3.21294 0.00112749 -5.75615e-07 1.31388e-09
                          -8.76855e-13 -1005.25 6.03474 );
    }
    transport
    {
        As              1.67212e-06;
        Ts              170.672;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** OpenFOAM v1906: 각 화학종을 구성하는 원소의 종류와 개수가 함께 정의된다.

> **Note:** OpenFOAM v2412: 각 화학종을 구성하는 원소의 종류와 개수가 함께 정의된다.

`thermodynamics` 하위 딕셔너리에는 열용량을 계산하기 위한 janaf 다항식 모델 계수가 저장되며, `transport` 하위 딕셔너리에는 점성 계산을 위한 Sutherland 모델 계수가 저장된다.

#### 3.1.4 system/ 디렉토리

controlDict 파일에서 `adjustTimeStep`을 `yes`로 설정하여 솔버가 `deltaT`를 무시하고 `maxCo`(최대 Courant 수)에 따라 자동으로 `deltaT`를 계산하도록 한다. `endTime`은 120(velocity-inlet-5에서의 속도를 기반으로 한 체적 체류 시간의 약 1배)으로 설정하고, `writeInterval`은 10으로 설정하여 10초마다 결과 파일을 저장하도록 한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
application     reactingFoam;

startFrom       startTime;
startTime       0;
stopAt          endTime;
endTime         120;
deltaT          1e-6;

writeControl    adjustableRunTime;
writeInterval   10;
purgeWrite      0;

writeFormat     ascii;
writePrecision  6;
writeCompression off;

timeFormat      general;
timePrecision   6;

runTimeModifiable true;

adjustTimeStep  yes;
maxCo           0.4;
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```
