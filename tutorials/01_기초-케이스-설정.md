## 1. 배경

### 1.1 전산유체역학이란?

전산유체역학(Computational fluid dynamics, CFD)이란, 컴퓨터에 기반한 시뮬레이션을 이용하여 유체 유동, 열전달, 화학 반응 등의 현상이 일어나는 시스템을 분석하는 학문으로, 산업·비산업 분야를 막론하고 다양한 영역에서 쓰이는 강력한 도구이다.

CFD 연구개발의 궁극적인 목표는 컴퓨터기반공학(Computer-aided engineering, CAE) 분야에 구조 해석 등과 동등한 수준의 시뮬레이션 기능을 제공하는 것이다. 그러나 유체 유동에 내재된 막대한 복잡성 때문에 지금까지는 다른 분야에 비해 뒤쳐져 있었다.

CFD에는 많은 장점이 있지만, 아직 물리 현상에 대한 실무적인 지식 없이 맹목적으로 사용할 수 있는 수준은 아니다. 또한, 적절한 조건을 설정하고 해석을 수행하는데 상당한 시간이 소요되기 때문에, 계산 자원이 비약적으로 발전했음에도 아직 실시간 계산에 활용할 수 있는 수준으로 성숙하지 않았다. 그럼에도 불구하고, CFD는 여타의 실험적 분석에 보완 역할을 해줄 수 있고, 실험적 연구에 함께 사용된다.

CFD 코드의 가장 일반적인 형태는 다음과 같다.

- 무료 공개 소스 코드
- 상업용 비공개 소스 코드

이 책은 C++로 작성된 무료 공개 소스 CFD 코드인 OpenFOAM의 활용 예제이다. OpenFOAM은 사용자가 소스 코드에 직접 접근하고 수정할 수 있어, 솔버를 직접 개발할 수도 있다.

### 1.2 OpenFOAM의 작업 순서

CFD 해석 절차는 유체 유동 문제를 해석할 수 있는 수치적 알고리즘을 중심으로 구성되고, 주로 전처리 - 솔버 실행 - 후처리의 세 가지 요소로 이루어진다.

#### 1.2.1 전처리

- 계산 영역(Computational domain) :  해석할 영역의 형상을 정의
- 격자 생성(mesh or grid generation) : 계산 영역을 작고 서로 겹치지 않는 셀(cell, element, or control volume)로 분할
- 해석하고자 하는 물리적·화학적 현상을 적절하게 표현하는 수학적 모델 및 서브모델의 선정
- 물성치(property) 정의
- 계산 영역의 경계와 맞닿은 격자 셀에 적절한 물리적, 화학적 경계 조건 부여

유체 유동 문제의 해석 변수(속도, 압력, 온도 등)는 각 격자 셀의 중심(cell center)에서 정의된다. CFD 해석 결과의 정확도는 계산 영역 내에 생성된 격자의 개수에 크게 의존한다. 최적의 격자는 불균일(non-uniform)한 형태인 경우가 많다. 즉, 인접한 두 지점 사이에서 해석 변수의 값의 변화가 큰 영역에는 격자를 더 조밀하게 생성하고 변화가 크지 않은 영역에는 더 성기게 생성하여, 격자의 개수는 최대한 덜 증가시키면서 주요 영역의 해석 정확도를 높일 수 있다.

#### 1.2.2 솔버 실행

수치해석 기법에는 최소한 네 종류의 서로 다른 방법이 있다. 유한차분법(finite difference method), 유한 요소법(finite element method), 스펙트럼 방법(spectral method), 유한체적법(finite volume method)이다. 대부분의 CFD 솔버는 유한체적법에 기초한 경우가 많으므로, 이 책도 유한체적법을 대상으로 한다. 유한체적법의 대략적인 해석 과정은 아래와 같다.

1. 계산 영역 내 모든 제어체적에 대해 질량, 에너지, 운동량 보존 방정식을 적분
2. 이산화 : 위의 적분 방정식을 대수 방정식 시스템으로 변환
3. 반복 계산(iteration)을 통한 대수 방정식의 풀이

이중 첫 번째 단계가 유한체적법이 다른 CFD 기법과 차별화되는 부분이다. 각 제어 체적에 대해 지배방정식을 적분하기 때문에, 운동량이나 엔탈피와 같은 물리 변수가 각각의 유한 체적 내에서 보존 법칙을 자동으로 만족하게 된다. 유한체적법 하에서는 사용된 수치적 알고리즘과 근본적인 보존 법칙 간의 관계가 명확하게 드러나기 때문에, 여러 CFD 기법 중 유한체적법이 가장 널리 사용되었고, 이해하기에도 가장 직관적이다.

#### 1.2.3 후처리

후처리는 해석이 완료된 후 그 해석 결과를 가시화하는 단계이다. 아래에 일반적으로 사용되는 후처리 도구를 나열하였다.

- 가시화를 위해 적절한 단면(section plane)을 정의
- 각종 물성치 및 물리량의 등치선(contour)
- 벡터장(vector field)
- 유선(streamline)
- 선 플롯(line plot)
- 수지(balance) 계산

후처리 전용 소프트웨어는 매우 다양하다. 유료 소프트웨어로는 Tecplot의 Tecplot, ANSYS의 CFD-POST 및 ensight 등이 있고, Paraview, SALOME 등의 무료 소프트웨어도 있다.

OpenFOAM은 ParaView를 실질적인 표준 후처리 도구로 사용한다. Paraview에는 OpenFOAM 해석 결과를 읽어들이는 전용 reader가 내장되어 있으며, OpenFOAM에는 케이스 디렉토리에 임시 `.foam` 파일을 만들고 ParaView를 실행해 주는 래퍼(wrapper) `paraFoam`이 내장되어 있어, 다른 후처리 도구에 비해 간편하게 사용할 수 있다.

### 1.3 `icoFoam`

`icoFoam`은 OpenFOAM에 내장된 솔버 중 하나로, 비뉴턴 유체의 비압축성, 층류 유동의 분석에 적절한 모델이다. PISO (pressure-implicit split operator) 알고리즘을 사용하여 비정상 상태의 압력-속도 연동을 반복 계산을 통해 해석한다. 각 반복 단계 마다 하나의 예측자(predictor)가 운동량 방정식의 근사치를 계산하고 두 개의 수정자(corrector)가 각각 속도와 압력의 근사치를 수정하여 업데이트한다.

## 2. 예제 개요

### 예제 구성

`icoFoam`을 이용하여 엘보 파이프 내부의 유동을 75초 간 해석한다. 3 종류의 격자에 대해 해석을 수행하여 격자 의존성을 확인한다.

- 삼각 격자 (OpenFOAM tutorial 내장)
- 성긴 사각 격자 (GAMBIT "elbow 2D" tutorial의 격자)
- 조밀한 사각 격자 (이전 단계 격자를 세분할(refinement)하여 사용)

### 목표

- OpenFOAM의 기본적인 설정 방법을 이해한다.
- 압력(p)과 유속(U)의 초기값 설정 방법을 이해한다.
- 경계면이 적절히 정의되었는지 확인한다. (GAMBIT에서 가져온 경계와 OpenFOAM에서 추가된 면)

### 데이터 처리

- 해석 결과를 ParaView로 가져오고, 스프레드시트 계산기를 이용하여 두 튜브 사이의 선을 따라 압력과 속력 분포를 도시한다. 세 조건의 시뮬레이션에 대해 반복한다.

#### 3.1.1 환경 설정

#### 3.1.2 예제 복사

터미널을 열고 아래 경로의 엘보 예제를 작업 디렉토리로 복사한다. 디렉토리 복사 등 리눅스의 터미널 실행 방법에 대해서는 부록 A를 참고하라.

```bash
$FOAM_TUTORIALS/incompressible/icoFoam/elbow
```

예를 들어,

```bash
cp -r $FOAM_TUTORIALS/incompressible/icoFoam/elbow /home/myWorkingDirectory
```

> **Note:** '$' 기호는 현재의 환경에서 OpenFOAM이 설치된 디렉토리로부터 예제 파일을 추출할 수 있도록 한다.

아래의 경로로부터 예제에서 사용할 GAMBIT 격자를 작업 디렉토리로 복사한다.

```bash
$FOAM_TUTORIALS/resources/geometry/elbow.msh
```

예를 들어,

```bash
cp -r $FOAM_TUTORIALS/resources/geometry/elbow.msh /home/myWorkingDirectory
```

#### 3.1.3 격자 파일 변환

GAMBIT으로 생성한 격자는 OpenFOAM과 바로 호환되지 않기 때문에, 아래의 명령어를 이용하여 OpenFOAM의 격자 파일로 변환해야 한다.

```bash
fluentMeshToFoam elbow.msh
```

OpenFOAM 형식으로 변환된 격자는 constant/ 디렉토리 하위에 새로 생성된 polyMesh/ 디렉토리에 저장된다. paraFoam 명령어를 이용해 paraview로 형상 및 격자를 확인한다.

```bash
paraFoam
```

OpenFOAM은 SI 단위계를 사용한다. GAMBIT, Solidworks 등 많은 CAD, 격자 생성 도구가 밀리미터 단위를 기본으로 사용하기 때문에, 밀리미터 단위를 기준으로 생성된 형상, 격자를 사용하는 경우에는 별도의 플래그를 이용해서 단위를 함께 변환해 주어야 한다. 각 명령어에서 사용할 수 있는 플래그는 명령어 뒤에 -help 플래그를 입력하여 확인할 수 있다.

```bash
fluentMeshToFoam -help
```

해당 명령어에 대해 사용 가능한 옵션과 각 옵션에 대한 간략한 설명이 출력된다.

```bash
//**************************************//
Usage: fluentMeshToFoam [OPTIONS] <Fluent mesh file>
options:
-2D <thickness>         // use when converting a 2-D mesh
                     // (applied before scale)
-case <dir>             // specify alternate case directory, 
                        // default is the cwd
-fileHandler <handler>      // override the fileHandler
-libs <(lib1 .. libN)>      // pre-load libraries
-noFunctionObjects  
                    // do not execute functionObjects
-scale <factor>     // geometry scaling factor 
                    // - default is 1
-writeSets          // write cell zones and patches as sets
-writeZones         // write cell zones as zones
-srcDoc             // display source code in browser
-doc                // display application documentation 
                    // in browser
-help               // print the usage

Using: OpenFOAM-7 (see https://openfoam.org)
Build: 7
//**************************************//
```

-scale 플래그는 격자의 길이 단위를 다른 단위계에서 SI 단위계로 변환할 때 사용한다. 밀리미터 단위로 생성된 격자를 미터 단위로 변환할 때는 -scale 0.001을 사용한다. -scale 플래그를 사용하지 않으면 1.0이 사용된다. 이 예제에서는 단위 변환 없이 격자 파일을 그대로 사용한다.

```bash
fluentMeshToFoam -scale 1.0
```

> **Note:** OpenFOAM으로 불러들인 격자는 반드시 3차원 격자여야 한다. 2차원 해석을 수행할 때는 3번째 방향으로 1개의 격자를 생성하여 3차원 데이터 구조를 만들어 사용한다. 1차원 해석의 경우에는 2번째 방향으로도 1개의 격자를 생성한다.

> **Note:** 격자 안에 내부 경계(internal boundary)가 있는 경우에는 `fluentMeshToFoam` 대신 `fluent3DMeshToFoam`을 사용한다.

#### 3.1.4 케이스 구조

OpenFOAM의 대부분의 케이스가 아래와 같은 형태의 케이스 구조(directory tree)를 기본으로 한다.

![케이스 구조](../figures/1_casestructure.png)

각 케이스 디렉토리에는 세 개의 주요 디렉토리(0/, constant/, system/)가 있다.

**0/ 디렉토리**

0/ 디렉토리는 시뮬레이션을 실행하기 위한 초기 조건이 저장된다. 0/ 디렉토리에 저장된 파일은 각각이 한 물리량의 초기 조건을 설정한다. 각 파일의 이름은 각 파일이 대표하는 물리량을 따라 지정된다. 예를 들어, T 파일은 온도의 초기 조건을 설정한다. 엘보 예제에서는 0/ 디렉토리에 2개의 파일만 저장되며, 각각 압력(p)과 유속(U)을 설정한다. 아래의 명령어로 압력의 초기 조건을 확인할 수 있다.

```bash
nano p
```

> **Note:** nano는 리눅스에 내장된 문서 편집기이며, Ctrl+X로 닫을 수 있다. 사용자의 편의에 따라 vim, emacs, kate, gedit 등 다른 문서 편집기를 이용해서 파일을 열 수 있다.

> **Note:** OpenFoam v2412: 0.orig/ 디렉토리의 사본을 생성하고 이름을 0으로 변경한다.
>
> ```bash
> cp -r 0.orig 0
> ```

```cpp
FoamFile
{
    version     2.0;
    format      ascii;
    class       volScalarField;
    object      p;
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
dimensions      [0 2 -2 0 0 0 0];

internalField   uniform 0;

boundaryField
{
    wall-4
    {
        type            zeroGradient;
    }
    velocity-inlet-5
    {
        type            zeroGradient;
    }
    velocity-inlet-6
    {
        type            zeroGradient;
    }
    pressure-outlet-7
    {
        type            fixedValue;
        value           uniform 0;
    }
    wall-8
    {
        type            zeroGradient;
    }
    frontAndBackPlanes
    {
        type            empty;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

`dimensions`는 해당 물리량의 SI 단위를 정의한다. 위에서는 p의 단위로 ($\mathrm{m^2 s^{-2}}$)을 입력하였다.

> **Note:** 전술한 것처럼, 이 예제에서 p의 단위는 압력의 단위($\mathrm{Pa}$)를 사용하지 않는다. 이는 OpenFOAM의 비압축성 유동 솔버에서 p를 압력이 아닌 "환산 압력(reduced pressure)", 즉 압력을 밀도로 나눈 값($p_{reduced}=p/\rho$)으로 정의하기 때문이다.

> **Note:** `dimensions` 행렬의 원소는 7개로, 순서대로 각각 질량(kilogram, $\mathrm{kg}$), 길이(meter, $\mathrm{m}$), 시간(second, $\mathrm{s}$), 온도(Kelvin, $\mathrm{K}$), 양(mole, $\mathrm{mol}$), 전류(ampere, $\mathrm{A}$), 빛의 세기(candela, $\mathrm{cd}$)를 의미한다.

`internalField`는 특정 물리량의 해석 영역 전체에 걸친 초기 장(field)를 설정한다. 균일(uniform), 비균일(non-uniform)의 두 종류로 설정할 수 있으며, uniform은 모든 격자에 동일한 값을 지정하고 non-uniform은 각 개별 격자 셀마다 초기값을 별도로 지정한다.

`boundaryField`는 각 경계면의 종류와 그 경계면에서의 해당 물리량의 값을 정의한다. OpenFOAM은 여러 종류의 경계 조건을 지원한다.

- **`zeroGradient`**: 해당 경계면에 0 구배 조건(Neumann 조건)을 적용한다.
- **`fixedValue`**: 해당 경계면에 고정값 조건(Dirichlet 조건)을 적용한다.
- **`empty`**: `empty`는 해석에 무관한 방향에 수직한 면에 적용하는 경계 조건이다. 예를 들어, 2차원 해석의 경우, 3번째 방향에 수직한 앞·뒷면에 `empty` 조건을 적용한다. 이 때, 한 방향에 수직한 두 개의 면을 하나의 경계면으로 이름을 설정하고 `empty` 조건을 한 번에 적용해야 한다.

> **Note:** `fixedValue` 경계 조건을 사용하면서 `$internalField`의 값과 동일한 값을 `value`로 설정하면 `zeroGradient` 경계 조건을 사용하는 것과 동일한 효과를 준다. 단, `zeroGradient`는 경계 조건을 암시적(implicitly)으로 적용하고, `fixedValue`를 `$internalField`와 함께 사용하면 해당 경계 조건을 명시적(explicitly)으로 적용한다는 차이점이 있다.

속도는 벡터량이기 때문에 U 파일은 3개의 요소를 이용해 각각 x, y, z방향의 속력을 정의한다. 이 예제는 2차원 해석이며 z방향으로는 계산이 실행되지 않기 때문에, z방향 속도는 항상 0이다. z방향에 수직한 경계면의 경계 조건은 `empty`로 설정되었다.

**constant/ 디렉토리**

constant/ 디렉토리는 보통 1개의 하위 디렉토리와 여러 파일로 구성된다. 하위 디렉토리 polyMesh는 격자 정보가 저장되는 디렉토리이고, 기타 파일들은 유체의 물성치나 해석에 고려될 물리·화학적 현상에 대한 정보를 포함한다. polyMesh/ 디렉토리의 boundary 파일은 각 경계면의 종류, patch 그룹, 격자면의 개수, ID 등 경계면에 대한 정보를 포함한다. (지면 문제로 인해, 이후의 소스 코드에서는 헤더 부분을 생략한다.)

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
6
{
    wall-4
    {
        type            wall;
        inGroups        List<word> 1(wall);
        nFaces          100;
        startFace       1300;
    }
    velocity-inlet-5
    {
        type            patch;
        nFaces          8;
        startFace       1400;
    }
...
    frontAndBackPlanes
    {
        type            empty;
        inGroups        List<word> 1(empty);
        nFaces          1836;
        startFace       1454;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** OpenFoam v1906: `inGroups`의 group 이름 앞에 자료형(`List<word>`)이 없음
> OpenFoam v2412: `inGroups`의 group 이름 앞에 자료형(`List<word>`)이 없음

경계면의 이름은 GAMBIT에서 설정된 이름과 동일해야 한다. 시작 셀 번호(`startFace`), 각 boundary를 구성하는 face cell의 개수(`nFaces`) 역시 boundary 파일에서 확인할 수 있다.

> **Note:** GAMBIT에는 `empty`형 경계 조건이 없다. 해석에 무관한 방향에 수직한 면은 `wall` 타입을 갖는 새로운 경계로 정의된다. GAMBIT으로 생성한 격자를 OpenFOAM 격자로 변환한 후에는 constant/polyMesh/boundary 파일을 열어서 해당 경계면의 `type`과 `inGroups`를 `wall`에서 `empty`로 변경해야 한다. 이 예제의 경우, 성긴 사각 격자와 조밀한 사각 격자를 OpenFOAM 형식으로 변환한 후 `frontAndBackPlanes` 경계면을 `empty`로 변경해야 한다.

transportProperties 파일에는 물성치가 정의되어 있으며, 각 물성치의 단위와 값을 확인, 수정할 수 있다. 예를 들어,

```cpp
nu      [0 2 -1 0 0 0 0]    0.01;
```

의 경우, `nu`는 유체의 동점성계수이고 그 값은 0.01 $\mathrm{m^2/s}$ 이다.

> **Note:** OpenFOAM v1906: 물성치의 단위는 나타나지 않는다.
> `nu`    0.01;

> **Note:** OpenFOAM v2412: 물성치의 단위는 나타나지 않는다.
> `nu`    0.01;

**system/ 디렉토리**

system/ 디렉토리는 솔버와 유한체적법의 설정이 저장되고, 다음의 3개 파일이 주요하다.

- **fvSchemes**: 지배 방정식의 각 항에 적용되는 이산화 기법이 정의된다.
- **fvSolution**: 압력-유속의 연동 기법, 각 물리량의 보존을 풀기 위해 사용되는 여러 수치적 기법, 각 물리량의 수렴성의 기준이 되는 허용 오차(tolerance)를 정의한다.
- **controlDict**: 시뮬레이션의 시작 시간(`startFrom`), 중지 시간(`stopAt`), 시간 간격(`deltaT`), 해석 결과의 저장 간격(`writeInterval`), 저장 포맷 (`writeFormat`), 해석 결과 저장 시 부동소수점의 정확도(`writePrecision`)가 정의된다. 솔버 실행 중 설정 파일에 변경 사항이 발생했을 때 이를 해석에 반영할 것인지의 여부(`runTimeModifiable`)도 `controlDict`에서 설정한다.

> **Note:** `writeFormat`을 `ascii`로 설정하면 저장된 해석 결과를 일반적인 문서 편집기로도 열 수 있다. 반면에 `binary`로 설정하면 해석 결과가 바이너리 형식으로 저장되어, 일반적인 문서 편집기로는 읽을 수 없다. 그러나 `binary`는 `ascii`에 비해 파일의 크기가 작기 때문에 파일을 더 빠르게 불러오거나 변환, 저장할 수 있다. 따라서 대규모 시뮬레이션을 실행하는 경우에는 `binary` 포맷을 활용하는 것이 작업 속도 면에서 유리하다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
{
application         icoFoam;
startFrom           latestTime;
startTime           0;
stopAt              endTime;
endTime             75;     // changed from 10 to 75 for 75 s 
                            // of simulation
deltaT              0.05;
writeControl        timeStep;
writeInterval       20;
purgeWrite          0;
writeFormat         ascii;
writePrecision      6;
writeCompression    off;
timeFormat          general;
timePrecision       6;
runTimeModifiable   true;
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** 이 시뮬레이션은 저장되어있는 해석 결과 중 마지막 시간대(`latestTime`)에서 시작한다. 저장된 해석 결과가 없는 경우에는 `startTime`에서 시작한다. 이 예제의 경우에는 0초이다.

### 3.2 시뮬레이션 실행

아래와 같이 사용하고자 하는 솔버의 이름을 입력하여 시뮬레이션을 실행한다.

```bash
icoFoam
```

> **Note:** 시뮬레이션을 실행하기 위해서는 솔버 실행 명령어(e.g. `icoFoam`)를 **메인 디렉토리의 내부**에서 실행해야 한다. 예를 들어, 이 예제의 경우에는 터미널에서 현재 위치(`pwd`)를 elbow/ 디렉토리로 이동한 후 명령어를 입력해야 한다. 그 하위 디렉토리나 다른 디렉토리에서 솔버 명령어를 실행하면 시뮬레이션이 정상적으로 실행되지 않는다.

### 3.3 후처리

#### 3.3.1 해석 결과 내보내기

OpenFOAM을 통해 만들어진 결과 파일은 사용하고자 하는 후처리 도구에 호환되는 형태로 변환해야 한다. ParaView는 VTK 포맷을 사용하므로, 아래와 같이 foam 데이터를 VTK 데이터로 변환한다.

```bash
foamToVTK
```

이 명령어 역시 케이스의 메인 디렉토리에서 입력해야 한다. 데이터를 ParaView에 맞게 변환한 후 ParaView를 실행한다.

```bash
paraview
```

> **Note:** OpenFOAM의 해석 결과를 VTK 포맷으로 변환하지 않고도 ParaView로 불러들일 수 있다. 메인 디렉토리에 빈 문서를 생성하고 이름을 \<someName\>.foam, 예를 들어, cavity\_result.foam으로 변경한다. 이후 아래의 명령어를 입력한다. 이 방식은 시뮬레이션 도중에 결과를 빠르게 확인해보거나 병렬 계산으로 분할된 케이스의 경우에 편리하게 사용할 수 있다.
> ```bash
> touch cavity_result.foam
> paraview cavity_result.foam &
> ```

> **Note:** OpenFOAM이 제공하는 래퍼 `paraFoam`을 이용하여 해석 결과를 VTK 포맷으로 변환하지 않고도 ParaView로 불러들일 수 있다.
>
> ```bash
> paraFoam &
> ```

> **Note:** 명령어의 끝에 & 기호를 추가하면 해당 명령어를 백그라운드로 실행하여, 작업이 완료되기를 기다리지 않고 터미널을 즉시 다시 사용할 수 있다.
