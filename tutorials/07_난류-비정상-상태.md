## 1. 배경

### 1.1 대와류모사법(large eddy simulation, LES)

대와류모사법(large eddy simulation, LES)에서는, 비교적 큰 와류는 주변 유동 공간의 형상에 영향을 받지만, 비교적 작은 와류는 좀 더 보편적(universal)이어서 주변 조건에 영향을 받지 않는는다고 가정한다. 따라서 큰 와류의 거동은 격자를 이용해 명시적(explicitly)으로 계산하고, 작은 와류의 영향은 아격자 스케일 모델(subgrid-scale, SGS model)로 모델링하여 처리한다.

이를 수학적으로 표현하면, 속도장에 필터 함수를 적용하여 직접 해석할 부분과 모델로 처리할 부분으로 분리하는 것과 같다. 해석할 부분에서는 큰 와류의 거동을 계산하고, 모델링할 부분에서는 작은 와류의 영향을 고려한다. 작은 와류의 영향은 SGS 모델을 통해 해석된 속도장에 포함된다. 필터링된 속도장 $\overline{u}_i\left(\vec{x}\right)$는 필터 커널 $G$와 속도장 $u$의 합성곱(convolution)으로 표현할 수 있다.

$$\overline{u}_i\left(\vec{x}\right) = \int G\left(\vec{x}-\vec{\xi}\right)u\left(\vec{\xi}\right)d\vec{\xi}$$

따라서,

$$u_i = \overline{u}_i + u_i'$$

여기서 $\overline{u}_i$는 속도의 해석 가능한(resolvable) 스케일 부분이고, $u_i'$는 아격자 스케일 부분이다. LES를 구현한 대부분의 상용 코드에서는 명시적인 필터링 단계를 거치지 않고, **격자 자체를 필터로 사용**한다. 필터링된 방정식은 비압축성 Navier-Stokes 방정식으로부터 도출할 수 있다.

$$\frac{\partial u_i}{\partial t} + u_j \frac{\partial u_i}{\partial x_j} = -\frac{1}{\rho} \frac{\partial p}{\partial x_i} + \frac{\partial}{\partial x_j} \left( v \frac{\partial u_i}{\partial x_j} \right)$$

필터링을 통해 분해한 변수 $u_i = \overline{u}_i + u_i'$와 $p_i = \overline{p}_i + p_i'$를 대입하고 정리하면 아래와 같이 해석 가능한 스케일에 대한 운동 방정식이 도출된다.

$$\frac{\partial \overline{u}_i}{\partial t} + \overline{u}_j \frac{\partial \overline{u}_i}{\partial x_j} = -\frac{1}{\rho} \frac{\partial \overline{p}}{\partial x_i} + \frac{\partial}{\partial x_j} \left( \overline{v} \frac{\partial \overline{u}_i}{\partial x_j} \right) + \frac{1}{\rho} \frac{\partial \tau_{ij}}{\partial x_j}$$

위 식은 필터링 연산과 미분 연산 간에 교환 법칙이 성립한다는 가정 하에 유도되었는데, 이 가정은 일반적으로 성립하는 것은 아니다. 미분 연산과 교환 법칙이 성립하는 필터 함수도 존재하기 때문에 이를 사용해도 된다. 그러나 교환 법칙의 성립 유무에 따른 오류는 대개 유의미할 만큼 크지는 않다고 여겨진다. $\frac{\partial \tau_{ij}}{\partial x_j}$는 아격자 스케일 응력(subgrid-scale, SGS stress)으로, 대류 항의 비선형성에 의해 발생한 추가 항이다. 즉,

$$u_j \frac{\partial u_i}{\partial x_j} \neq \overline{u}_j \frac{\partial \overline{u}_i}{\partial x_j}$$

이므로,

$$\tau_{ij} = \overline{u}_i\overline{u}_j - \overline{u_i u_j}$$

이다.

아격자 스케일의 유동장에 대해서도 비슷한 방정식을 유도할 수 있다. 아격자 스케일 난류 모델은 주로 Boussinesq 가정을 채택하여 SGS (편차, deviatoric)응력을 아래와 같이 계산한다.

$$\tau_{ij}-\frac{1}{3}\tau_{kk}\delta_{ij} = -2\mu_t \overline{S}_{ij}$$

여기서 $\overline{S}_{ij}$는 해석 가능한 스케일의 변형률 텐서로, 아래와 같이 정의된다.

$$\overline{S}_{ij} = \frac{1}{2} \left( \frac{\partial \overline{u}_i}{\partial x_j} + \frac{\partial \overline{u}_j}{\partial x_i} \right)$$

$\nu_t$는 아격자 스케일의 난류 점성이다. 이를 필터링된 Navier-Stokes 방정식에 대입하면 다음과 같다.

$$\frac{\partial \overline{u}_i}{\partial t} + \overline{u}_j \frac{\partial \overline{u}_i}{\partial x_j} = -\frac{1}{\rho} \frac{\partial \overline{p}}{\partial x_i} + \frac{\partial}{\partial x_j} \left( \left[ \nu + \nu_t \right] \frac{\partial \overline{u}_i}{\partial x_j} \right)$$

여기서 비압축성 조건을 이용해 식을 단순화하였고, 압력은 대각합 항(trace term)인 $\tau_{kk}\delta_{ij}/3$을 포함한다.

### 1.2 Smagorinsky-Lilly 모델

Smagorinsky-Lilly 모델은 LES의 아격자 스케일 모델 중 가장 간단한 0방정식(대수식) 형태로, 아래와 같이 표현된다.

$$\tau_{ij}-\frac{1}{3}\tau_{kk}\delta_{ij} = -2 \left(C_s \Delta \right)^2 |\overline{S}| S_{ij}$$

와점성은 아래와 같이 정의된다.

$$\mu_{sgs} = \rho \left( C_s \Delta \right)^2 |\overline{S}|$$

$\Delta$는 필터의 너비를 뜻하며, 일반적으로 아래와 같이 계산한다.

$$\Delta = \left( \text{volume} \right)^{1/3}$$

변형률 텐서의 크기는 아래와 같이 계산한다.

$$\overline{S} = \sqrt{2S_{ij}S_{ij}}$$

유효 점성은 유체의 점성과 와점성의 합으로 계산한다.

$$\mu_{eff} = \mu_{mol} + \mu_{sgs}$$

$C_s$는 Smagorinsky 상수로, 0.1--0.2의 값을 갖는다.

## 2. 예제 개요

### 예제 구성

`pisoFoam`을 이용해 후향 계단(backward facing step)의 난류 유동을 0.2초 간 해석한다. 아래의 난류 모델을 사용하여 결과를 비교한다.

- Smagorinsky (LES)
- kEqn (LES)
- kEpsilon (RAS)

### 목표

- 난류 모델을 이해한다.
- 정상 상태와 비정상 상태 시뮬레이션의 차이를 이해한다.
- 적절한 난류 모델을 선정한다.

### 데이터 처리

- 두 개로 분리된 등치선 플롯에 세 개의 다른 시간대에서 U와 난류 점성의 계산 결과를 플롯한다. [예제 6](./06_난류-정상-상태.md)에서 수행한 정상 상태 시뮬레이션의 결과와 비교한다.

## 3. 해석

### 3.1 전처리

#### 3.1.1 예제 복사

아래의 예제를 작업 디렉토리로 복사한다.

```bash

```

#### 3.1.2 0/ 디렉토리

난류 모델의 초기 조건과 경계 조건을 설정한다.

> **Note:** 각 난류 모델 별로 모델 파라미터 및 입력 조건이 다르기 때문에, 수정해야하는 파일이 서로 다르다. 예제 6을 참고한다.

#### 3.1.3 constant/ 디렉토리

난류 모델의 종류는 turbulenceProperties 파일에서 설정한다. `simulationType`을 `LES` 또는 `RAS`로 변경한다. 그 후 각 모델에 해당하는 하위 딕셔너리를 정의한다. 아래는 kEqn 모델을 이용한 LES 해석을 위한 turbulenceProperties 파일이다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
simulationType          LES;
LES
{
    LESModel            kEqn;
    turbulence          on;
    printCoeffs         on;
    delta               cubeRootVol;

    dynamicKEqnCoeffs
    {
        filter          simple;
    }
    cubeRootVolCoeffs
    {
        deltaCoeff      1;
    }

    PrandtlCoeffs
    {
        delta           cubeRootVol;
        cubeRootVolCoeffs
        {
            deltaCoeff  1;
        }
        SmoothCoeffs
        {
            delta       cubeRootVol;
            cubeRootVolCoeffs
            {
                deltaCoeff  1;
            }
            maxDeltaRatio   1.1;
        }
        Cdelta              0.158;
    }
    vanDriestCoeffs
    {
        delta           cubeRootVol;
        cubeRootVolCoeffs
        {
            deltaCoeff      1;
        }
        smoothCoeffs
        {
            delta       cubeRootVol;
            cubeRootVolCoeffs
            {
                deltaCoeff  1;
            }
            maxDeltaRatio   1.1;
        }
        Aplus               26;
        Cdelta              0.158;
    }
    smoothCoeffs
    {
        delta               cubeRootVol;
        cubeRootVolCoeffs
        {
            deltaCoeff      1;
        }
        maxDeltaRatio       1.1;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** `kEqn`은 1방정식 LES SGS 모델로, 아격자 스케일 난류 운동에너지에 대한 수송 방정식을 풀어 아격자 스케일의 난류 점성을 계산한다. 아격자 스케일 난류 점성을 대수식으로 모델링하는 Smagorinsky-Lilly 모델과 구분된다.

> **Note:** OpenFOAM v2412: 기본 SGS 모델이 `dynamicKEqn`으로 변경되었다. `dynamicKEqn`은 `kEqn`을 기본으로 하되 Germano 절차를 적용하여 모델 상수를 동적으로 결정한다.

### 3.3 후처리

해석 결과는 아래와 같다.

kEpsilon 모델을 사용한 경우, 0.2초의 해석 결과가 정상 상태 시뮬레이션 결과와 유사하다. 따라서 정상 상태에 도달했다고 볼 수 있다. 그러나 다른 모델을 사용한 해석 결과는 정상 상태에 도달하지 않고 계속 변동하기 때문에, 정상 상태의 결과를 얻기 위해서는 각 시간 단계에서의 해석 결과의 평균을 구해야 한다.

kEpsilon 등의 RAS 모델은 난류 변수의 값을 계산할 떄 평균을 사용한다. 그러나 LES는 기본적으로 평균화 기능을 가지고 있지 않다. 그렇기 때문에 LES 시뮬레이션에는 더 조밀한 격자와 더 작은 시간 단계를 사용해야 한다. 등치선 등의 해석 결과는 적절한 시간 동안의 평균값으로 평가해야 한다. (이 예제에서는 수행하지 않았다.)
