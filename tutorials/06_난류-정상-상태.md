## 1. 배경

### 1.1 난류 모델링

공학적 유체 현상에는 난류 유동이 많다. 난류는 매우 시간 의존적 현상이며, 다양한 크기의 와류(eddy)를 갖는다. 이 와류를 직접 해석(direct numerical simulation, DNS)하여 난류 유동장의 분포를 완전히 계산할 수도 있지만, 이는 계산 비용이 막대하게 소요되어, 형상이 복잡한 공학적 문제에 항상 적용할 수 있는 것은 아니다. 따라서 대부분의 공학적 응용에서는 난류를 완전히 해석하지 않고 모델로 대체하여 유동장을 해석한다.

난류 모델링의 핵심은 난류 현상에 대한 지배 방정식의 해를 단순화하는 평균화(averaging)이다. 계산 자원의 제한으로, 모든 물리적 현상을 원하는 만큼의 격자 해상도 및 시간 해상도로 해석할 수 없다. 따라서 현재 사용 가능한 격자가 해석할 수 없는 작은 스케일의 현상에 대해서는 모델을 적용한다.

아래는 대표적인 난류 모델을 나타낸다.

- RANS 기반 모델
    - 선형 와점성(eddy viscosity) 모델
        - 대수 모델
        - 1방정식 또는 2방정식 모델
    - 비선형 와점성 모델, 대수 응력 모델
    - Reynolds 응력 수송 모델
- 대와류모사법(large eddy simulation, LES)
- 분리와류모사법(detached eddy simulation, DES) 등의 복합 모델

이 예제에서는 RANS 기반 모델을 자세히 설명한다. 다음 예제에서 Smagorinsky-Lilly 서브모델을 이용한 LES 해석에 대해 다룬다.

### 1.2 RANS 기반 난류 모델

뉴턴 유체의 유동에 대한 지배 방정식은 아래와 같다.

- 질량 보존

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot \left( \rho \mathbf{\widetilde{u}} \right) = 0
$$

- 운동량 보존 (Navier-Stokes 방정식)

$$
\frac{\partial \rho \mathbf{\widetilde{u}_i}}{\partial t} + \nabla \cdot \left( \rho \mathbf{\widetilde{u}_i} \mathbf{\widetilde{u}} \right) = -\frac{\partial \widetilde{p}}{\partial x_i} + \nabla \cdot \left( \mu \nabla \mathbf{\widetilde{u}_i} \right) + \mathbf{\widetilde{S}_{M,i}}
$$

- 임의의 수동적 스칼라량 $e$의 보존

$$
\frac{\partial \rho \widetilde{e}}{\partial t} + \nabla \cdot \left( \rho \widetilde{e} \mathbf{\widetilde{u}} \right) = \nabla \cdot \left( k \nabla \widetilde{T} \right) + \widetilde{S}_e
$$

> **Note:** 운동량 방정식은 간결하게 표현하기 위해 하첨자를 사용하였다. $i$는 방향을 의미하며, $i=1,2,3$은 각각 x, y, z 방향을 의미한다.

난류 문제의 해결 방법 중 하나는 Reynolds 분해(decomposition)를 통해 스케일의 개수를 (무한대에서 한두 개 정도로) 줄이는 것이다. Reynolds 분해란, 어떤 순간(instantaneous) 물리량 $\widetilde{\varphi}$를 그 평균(average) $\varPhi$와 요동(fluctuation) $\varphi$의 합으로 표현하는 것이다. 즉, $\widetilde{\varphi}=\varPhi+\varphi$이다. Navier-Stokes 방정식에 Reynolds 분해를 적용하면 Reynolds-평균 Navier-Stokes (Reynolds-averaged Navier Stokes, RANS) 방정식을 얻는다.

- 평균 질량 보존

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot \left( \rho \mathbf{U} \right) = 0
$$

- 평균 운동량 보존 (RANS 방정식)

$$
\frac{\partial \rho U_i}{\partial t} + \nabla \cdot \left( \rho U_i \vec{U} \right) = -\frac{\partial P}{\partial x_i} + \nabla \cdot \left( \mu \nabla U_i \right) -\left( \frac{\partial \left( \rho \overline{uu}_l \right)}{\partial x} + \frac{\partial \left( \rho \overline{vu}_l \right)}{\partial y} + \frac{\partial \left( \rho \overline{wu}_l \right)}{\partial z} \right) + S_{M,i}
$$

- 평균 수동적 스칼라량 보존

$$
\frac{\partial \rho E}{\partial t} + \nabla \cdot \left( \rho E \vec{U} \right) = \nabla \cdot \left( k \nabla T \right) + S_e
$$

> **Note:** Reynolds 분해의 특수한 성질 중 하나는, 요동 성분의 평균이 항상 0이라는 것이다. 예를 들어, $\overline{u}=\overline{v}=\overline{w}=0$이다. 위의 평균 보존 방정식을 유도할 때 이 성질이 사용된다.

그러나, Reynolds 분해를 적용함에 따라, 난류 응력($\rho \overline{uu}$, $\rho \overline{vu}$, $\rho \overline{wu}$, $\rho \overline{uv}$, $\rho \overline{vv}$, $\rho \overline{wv}$, $\rho \overline{uw}$, $\rho \overline{vw}$, $\rho \overline{ww}$), 난류 선속($\rho \overline{ue}$, $\rho \overline{ve}$, $\rho \overline{we}$) 등의 새로운 미지수가 발생한다. 방정식의 개수보다 미지수의 개수가 더 많기 때문에, RANS 방정식은 닫힌 형태(closed form)가 아니다. 그러면 어떻게 해야 RANS 방정식을 닫힌 형태로 정식화할 수 있는가? 이 문제를 **난류 종결 문제**(turbulence closure problem)라 하고, 새로운 미지수를 모델링하는 추가적인 방정식을 도출하는 것을 난류 모델링이라 한다.

추가된 미지수는 총 9개로, Reynolds 응력 항이 6개($\rho \overline{uv} = \rho \overline{vu}$, ...), 난류 선속 항이 3개다. 온도 등의 스칼라량이 하나 포함된 가장 단순한 난류 유동 문제의 경우에도 $u$, $v$, $w$, $p$, $T$를 포함하여 총 14개의 미지수를 계산해야 한다.

추가된 미지수를 모델링하는 여러 방법 중 하나는 난류 응력과 난류 선속에 대한 편미분방정식을 계산하는 것이다. 아래에 대표적인 난류 모델을 단순한 것부터 복잡한 순으로 나열하였다.

- 대수(0방정식) 모델 : 혼합 길이 (1차)
- 1방정식 모델 : k모델, $\mu_t$모델 (1차)
- 2방정식 모델 : k-$\varepsilon$, k-kl, k-$\omega$, low-Re k-$\varepsilon$ (1차)
- 대수 응력 모델(Algebraic stress model, ASM) (2차)
- Reynolds 응력 모델(Reynolds stress model, RSM) (2차)
- 0방정식 모델

OpenFOAM은 난류 유동의 해석을 크게 RAS와 LES의 두 종류로 구분한다. RAS 해석은 RANS 기반 모델을 이용한 것으로, 이 예제에서 다루는 내용이다. 다음 예제에서는 LES 해석을 설명하고, RAS와 LES의 해석 결과를 비교한다.

## 3. 해석

### 3.1 전처리

#### 3.1.1 예제 복사

아래의 예제를 작업 디렉토리로 복사한다.

```bash
$FOAM_TUTORIALS/incompressible/simpleFoam/pitzDaily
```

#### 3.1.2 0/ 디렉토리

난류 모델을 선택한 경우에는 각 난류 변수의 값과 경계 조건이 각 파일에 적절히 설정되어야 한다. 예를 들어, kEpsilon 모델을 선택했을 때는 k와 epsilon 파일을 수정한다. 아래는 0/ 디렉토리의 epsilon 파일이다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
dimensions          [0 2 -3 0 0 0 0];

internalField       uniform 14.855;

boundaryField
{
    inlet
    {
        type        fixedValue;
        value       uniform 14.855;
    }
    outlet
    {
        type        zeroGradient;
    }
    upperWall
    {
        type        epsilonWallFunction;
        value       uniform 14.855;
    }
    lowerWall
    {
        type        epsilonWallFunction;
        value       uniform 14.855;
    }
    frontAndBack
    {
        type        empty;
    }
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

> **Note:** 각 난류 모델 별 난류 변수 설정 파일을 아래에 나열하였다.
>
> - laminar: 없음
> - kEpsilon (RAS): k, epsilon
> - kOmega (RAS): k, omega
> - LRR (RAS): k, epsilon, R
> - Smagorinsky (LES): nuSgs
> - kEqn (LES): k, nuSgs
> - SpalartAllmaras (LES): nuSgs, nuTilda

k, epsilon, nuTilda 등의 파일은 이미 만들어져 있어서 기존의 예제에서 복사하여 사용할 수 있으나, R, nuSgs 등과 같이 파일을 사용자가 직접 생성해야하는 경우도 있다. 해당 양식은 OpenFOAM의 구 버전에서 찾을 수 있다.

> **Note:** R 파일을 생성하는 경우, 먼저 constant/ 디렉토리의 turbulenceProperties 파일에서 `simulationType`을 `RAS`로, `RASmodel`을 `kEpsilon`으로 변경한다. 그 후 아래 명령어를 실행하면 0/ 디렉토리에 R 파일이 생성된다.
> ```bash
> simpleFoam -postProcess -func R
> ```

> **Note:** OpenFOAM v1906: 위 방법을 따르면 R 대신 turbulenceProperties:R 파일이 생성된다. 생성된 파일의 이름을 R로 변경하여 사용한다.

#### 3.1.3 constant/ 디렉토리

먼저 turbulenceProperties 파일에서 `simulationType`을 `RAS`, `LES`, `laminar` 등으로 설정한다. 그 후 각 모델에 해당하는 하위 딕셔너리를 정의한다. 이 예제의 경우, `RAS`에서 난류 모델을 설정하고, `turbulence`와 `printCoeffs`의 상태를 `on`으로 변경한다.

> **Note:** `laminar`를 선택한 경우에는 `turbulence`와 `printCoeffs`의 상태를 `off`으로 변경한다.

#### 3.1.4 system/ 디렉토리

> **Note:** 정상 상태 솔버를 실행하는 경우, controlDict 파일에 저장된 변수의 의미가 달라진다. `endTime`은 시간이 아니라 반복 계산의 횟수(number of iteration)를 가리킨다. `deltaT`는 시간 간격이 아니라 반복 간격, 즉 반복 수의 증분을 의미하므로, 반드시 1로 설정해야 한다.

LRR 모델을 사용하는 경우, fvSchemes 파일에서 새로운 변수인 R에 대한 이산화 기법을 정의해야 한다.

```cpp
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
ddtSchemes
{
    default             steadyState;
}
gradSchemes
{
    default             Gauss linear;
}
divSchemes
{
    default             none;
    div(phi,U)          bounded Gauss linearUpwind grad(U);
    div(phi,k)          bounded Gauss linearUpwind 1;
    div(phi,epsilon)    bounded Gauss linearUpwind 1;
    div(phi,omega)      bounded Gauss linearUpwind 1;
    div(phi,v2)         bounded Gauss linearUpwind 1;
    div(phi,R)          bounded Gauss linearUpwind 1;
    div(R)              Gauss linear;
    div((nuEff*dev2(T(grad(U)))))       Gauss linear;
    div(nonlinearStress)                Gauss linear;
}
laplacianSchemes
{
    default     Gauss linear corrected;
}
interpolationSchemes
{
    default             linear;
}
snGradSchemes
{
    default             corrected;
}
wallDist
{
    Method              meshWave;
}
// * * * * * * * * * * * * * * * * * * * * * * * * * * * * *//
```

또한, fvSolution 파일 역시 새로운 변수 R의 해석 기법을 정의해야 한다. 이 예제에서는 다른 변수(U, k, epsilon, omega)와 동일한 솔버를 사용한다.
